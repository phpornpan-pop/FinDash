# สมุดบัญชี: สินทรัพย์ · หนี้สิน · ความมั่งคั่งสุทธิ

เว็บแอปบันทึกสินทรัพย์/หนี้สินรายไตรมาส พร้อมเป้าหมายและ DCA ต่อสินทรัพย์
ผู้ดูแลต่อรายการ โมดูลประกันชีวิต/สุขภาพ/อุบัติเหตุ/บ้าน/รถ และวางแผนภาษีรายปี

โปรเจกต์นี้เป็น React + Vite ธรรมดา รันเป็นเว็บสถิต (static site) ได้ทุกที่
และรองรับ **Supabase เป็นฐานข้อมูล** พร้อมระบบล็อกอิน — แต่ละคนที่ใช้เว็บนี้
จะมีข้อมูลของตัวเองแยกกันอย่างปลอดภัย เหมาะกับการแชร์ลิงก์ให้เพื่อน/ครอบครัวใช้ร่วมกัน

---

## 1. รันในเครื่องตัวเอง

ต้องมี [Node.js](https://nodejs.org) ติดตั้งไว้ก่อน (แนะนำ v18 ขึ้นไป)

```bash
npm install
npm run dev
```

เปิด http://localhost:5173 จะเห็นแอปทันที ตอนนี้ข้อมูลจะถูกเก็บใน
localStorage ของเบราว์เซอร์เท่านั้น (ยังไม่เชื่อม Supabase ไม่ต้องล็อกอิน) —
ไปทำขั้นตอนที่ 2 ต่อเพื่อให้ข้อมูลถาวร มีระบบล็อกอินแยกคน และดูได้จากหลายเครื่อง

---

## 2. ตั้งค่า Supabase เป็นฐานข้อมูล (มีระบบล็อกอินในตัว)

### 2.1 สร้างโปรเจกต์ Supabase
1. ไปที่ [supabase.com](https://supabase.com) → Sign up (ฟรี ใช้บัญชี GitHub ก็ได้)
2. กด **New project** ตั้งชื่อโปรเจกต์อะไรก็ได้ ตั้งรหัสผ่านฐานข้อมูล (เก็บไว้เผื่อใช้)
   เลือก Region ใกล้ที่สุด (เช่น Singapore) → กด Create project (รอประมาณ 1-2 นาที)

### 2.2 สร้างตารางข้อมูล
1. ในเมนูซ้าย กด **SQL Editor** → New query
2. คัดลอกเนื้อหาทั้งหมดจากไฟล์ `supabase/schema.sql` ในโปรเจกต์นี้ไปวาง
3. กด **Run** — จะได้ตาราง `ledger_data` พร้อมกฎความปลอดภัย (Row Level Security)
   ที่ทำให้แต่ละคนเห็น**เฉพาะข้อมูลของตัวเอง**เท่านั้น

### 2.3 เอา URL และ Key มาใส่ในแอป
1. ในเมนูซ้าย กด **Project Settings** (รูปเฟือง) → **API**
2. คัดลอกค่า 2 ค่านี้:
   - **Project URL** (หน้าตาเช่น `https://xxxxx.supabase.co`)
   - **anon public** key (กุญแจยาวๆ ใต้หัวข้อ Project API keys)
3. คัดลอกไฟล์ `.env.example` เป็นไฟล์ชื่อ `.env` แล้วใส่ค่า:
   ```
   VITE_SUPABASE_URL=https://xxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=xxxxxxxxxxxxxxxxxxxxx
   ```
4. รัน `npm run dev` ใหม่ — ตอนนี้แอปจะขึ้นหน้าล็อกอิน/สมัครสมาชิกก่อนเข้าใช้งาน

### 2.4 (แนะนำ) ปิดการยืนยันอีเมล เพื่อให้สมัครแล้วใช้ได้ทันที
ปกติ Supabase จะบังคับให้ผู้สมัครใหม่กดยืนยันลิงก์ในอีเมลก่อนถึงจะล็อกอินได้
ถ้าอยากให้เพื่อนสมัครแล้วใช้งานได้เลยไม่ต้องเช็คอีเมล:
1. ในเมนูซ้าย กด **Authentication** → **Providers** → **Email**
2. ปิดสวิตช์ **"Confirm email"**
3. กด Save

> ข้อควรระวัง: การปิดยืนยันอีเมลทำให้สมัครง่ายขึ้น แต่ใครก็ใช้อีเมลปลอมสมัครได้
> เหมาะกับการใช้งานกันเองในกลุ่มเพื่อน/ครอบครัวที่ไว้ใจกัน

**หมายเหตุการทำงาน:** ข้อมูลของแต่ละคนถูกเก็บแยกกันในตาราง `ledger_data`
โดยผูกกับ user ID ที่ได้จากการล็อกอิน ไม่มีทางเห็นข้อมูลของคนอื่นได้เลย
(บังคับด้วย Row Level Security ในระดับฐานข้อมูล ไม่ใช่แค่ในโค้ดฝั่งหน้าเว็บ)

## 3. อัปโหลดขึ้น GitHub

```bash
git init
git add .
git commit -m "Initial commit: networth ledger app"
```

ไปสร้าง repository เปล่าบน https://github.com/new (อย่าติ๊กเพิ่ม README/.gitignore
เพราะเรามีอยู่แล้ว) แล้วรัน:

```bash
git remote add origin https://github.com/<username>/<repo-name>.git
git branch -M main
git push -u origin main
```

**ข้อควรระวัง:** ไฟล์ `.env` ที่ใส่ค่า Supabase ไว้จะไม่ถูกอัปโหลด
(อยู่ใน `.gitignore` แล้ว) ถ้าจะ deploy ขึ้นเว็บจริง ต้องตั้งค่า environment
variables `VITE_SUPABASE_URL` และ `VITE_SUPABASE_ANON_KEY` แยกต่างหากบน
แพลตฟอร์มที่ deploy (ดูข้อ 4)

หรือถ้าไม่สะดวกใช้ `git` ในเครื่อง ใช้วิธีลากไฟล์ทั้งโฟลเดอร์อัปโหลดผ่านหน้าเว็บ
GitHub ได้เหมือนกัน (กด "uploading an existing file" ในหน้า repo เปล่า)
แต่ต้องลาก**ทั้งโฟลเดอร์** `src`, `supabase` เข้าไปให้โครงสร้างไฟล์ถูกต้อง

---

## 4. Deploy ให้เข้าเว็บได้จริง

รองรับทั้ง **Vercel**, **Netlify**, และ **Render** (Static Site) — ตั้งค่าคล้ายกัน:

1. สมัคร/ล็อกอินด้วยบัญชี GitHub
2. สร้างโปรเจกต์ใหม่ เลือก repo ที่เพิ่ง push
3. ตั้งค่า build:
   - **Build Command:** `npm install && npm run build` (หรือ `npm run build` เฉยๆ ก็ได้ถ้าแพลตฟอร์ม install ให้อัตโนมัติ)
   - **Publish/Output Directory:** `dist`
4. ตั้งค่า **Environment Variables**:
   - `VITE_SUPABASE_URL` = URL จากข้อ 2.3
   - `VITE_SUPABASE_ANON_KEY` = anon key จากข้อ 2.3
5. กด Deploy — รอ 1-3 นาที จะได้ลิงก์เว็บที่ใช้งานได้จริง

**แชร์ลิงก์ให้เพื่อนได้เลย** — แต่ละคนกดสมัครสมาชิกด้วยอีเมลตัวเอง แล้วจะมี
ข้อมูลสินทรัพย์/หนี้สิน/ประกัน/ภาษีเป็นของตัวเอง แยกจากคนอื่นโดยสมบูรณ์

## โครงสร้างไฟล์

```
networth-ledger/
├── src/
│   ├── App.jsx                  ← ตัวแอปทั้งหมด (UI + logic + auth gate)
│   ├── main.jsx                  ← จุดเริ่มต้น React
│   ├── index.css                 ← Tailwind
│   └── lib/
│       ├── supabaseClient.js     ← สร้าง Supabase client จาก .env
│       ├── auth.js               ← ฟังก์ชันสมัคร/ล็อกอิน/ล็อกเอาท์
│       └── storage.js            ← อ่าน/เขียนข้อมูลจาก Supabase + fallback localStorage
├── supabase/
│   └── schema.sql                ← SQL สร้างตาราง + Row Level Security (ข้อ 2.2)
├── google-apps-script/
│   └── Code.gs                    ← (ของเดิม) ทางเลือกแบบ Google Sheets ไม่มีระบบล็อกอิน
│                                    เก็บไว้เผื่ออยากใช้แทน Supabase สำหรับผู้ใช้คนเดียว
├── .env.example
├── package.json
└── ...
```

## ทางเลือกอื่นนอกจาก Supabase
- **ไม่เชื่อมอะไรเลย** — ปล่อย `.env` ว่างไว้ ใช้ localStorage อย่างเดียว ง่ายสุด
  แต่ข้อมูลอยู่แค่เบราว์เซอร์เครื่องนั้น ไม่มีระบบล็อกอิน
- **Google Sheets** — ของเดิมที่เคยตั้งไว้ (ไฟล์ `google-apps-script/Code.gs` ยังอยู่)
  เหมาะกับผู้ใช้คนเดียวที่อยากดูข้อมูลเป็นตารางใน Sheets แต่**ไม่มีระบบแยกผู้ใช้**
  ถ้าจะกลับไปใช้แบบนี้ต้องแก้โค้ดใน `src/lib/storage.js` กลับไปเรียก Web App URL แทน

## ข้อจำกัดที่ควรรู้
- Supabase ฟรี tier เพียงพอสำหรับใช้งานส่วนตัว/กลุ่มเล็กสบายๆ (500MB ฐานข้อมูล,
  ผู้ใช้ auth ไม่จำกัดจำนวนในแผนฟรี ณ ตอนที่เขียนนี้)
- ถ้าปิด "Confirm email" ไว้ ใครก็สมัครด้วยอีเมลอะไรก็ได้ ไม่มีการยืนยันตัวตนจริงจัง
- รหัสผ่านที่ผู้ใช้ตั้งจะถูกเก็บ/จัดการโดย Supabase Auth (เข้ารหัสตามมาตรฐาน)
  ไม่ได้ผ่านโค้ดของเราโดยตรง
